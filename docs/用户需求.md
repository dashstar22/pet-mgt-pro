# 宠物领养平台 — 用户需求文档

## 一、项目背景

随着城市宠物数量的增长，流浪动物救助和宠物领养成为社会关注的热点问题。传统的宠物领养方式依赖线下门店或社交媒体，存在信息分散、审核流程不规范、匹配效率低等问题。

宠物领养平台旨在构建一个线上宠物领养管理系统，通过信息化手段连接宠物救助机构与领养用户，提供宠物信息管理、领养申请审核、AI智能匹配等功能，提升宠物领养的效率和透明度。

本项目是在《Web开发技术II》课程项目基础上的升级改造，将原有的 Spring Boot + Thymeleaf 服务端渲染架构改造为前后端分离架构（Spring Boot REST API + Vue3），以符合现代 Web 应用开发的主流范式。

---

## 二、用户角色

| 角色 | 说明 | 核心需求 |
|------|------|---------|
| **游客** | 未登录的浏览用户 | 浏览宠物列表、查看宠物详情、注册账号 |
| **普通用户** | 已注册登录的用户 | 申请领养、查看申请状态、AI 匹配、AI 问答、管理个人信息 |
| **管理员** | 系统管理员 | 管理宠物信息、管理品种、审核领养申请、管理用户 |

## 三、功能需求

### 3.1 用户认证模块

| 编号 | 功能 | 描述 | 角色 |
|------|------|------|------|
| F-AUTH-01 | 用户注册 | 填写用户名、密码、邮箱完成注册 | 游客 |
| F-AUTH-02 | 用户登录 | 用户名+密码登录，获得 JWT Token | 游客 |
| F-AUTH-03 | 个人中心 | 查看和修改个人信息、头像 | 普通用户 |
| F-AUTH-04 | 修改密码 | 修改登录密码 | 普通用户 |
| F-AUTH-05 | 退出登录 | 清除 Token，返回首页 | 普通用户/管理员 |

### 3.2 宠物浏览模块

| 编号 | 功能 | 描述 | 角色 |
|------|------|------|------|
| F-PET-01 | 首页展示 | 展示最新上架的宠物列表（8 条） | 所有用户 |
| F-PET-02 | 宠物列表 | 分页展示宠物，支持按品种、状态筛选 | 所有用户 |
| F-PET-03 | 宠物详情 | 查看宠物详细信息（年龄、性格、健康状况、图片等） | 所有用户 |
| F-PET-04 | 宠物搜索 | 按宠物名称关键字搜索 | 所有用户 |

### 3.3 领养申请模块

| 编号 | 功能 | 描述 | 角色 |
|------|------|------|------|
| F-APP-01 | 提交申请 | 选择宠物后填写申请信息并提交 | 普通用户 |
| F-APP-02 | 我的申请 | 查看自己所有的领养申请记录及状态 | 普通用户 |
| F-APP-03 | 取消申请 | 取消处于"待审核"状态的申请 | 普通用户 |
| F-APP-04 | 删除记录 | 删除已完成或被拒绝的申请记录 | 普通用户 |

### 3.4 AI 智能模块

| 编号 | 功能 | 描述 | 角色 |
|------|------|------|------|
| F-AI-01 | AI 宠物匹配 | 用户描述需求偏好，AI 推荐最匹配的宠物 | 普通用户 |
| F-AI-02 | 匹配历史 | 查看历史匹配记录 | 普通用户 |
| F-AI-03 | AI 养宠问答 | 回答养宠相关问题（饮食、健康、训练等） | 普通用户 |

### 3.5 管理后台模块

| 编号 | 功能 | 描述 | 角色 |
|------|------|------|------|
| F-ADM-01 | 仪表盘 | 后台首页统计数据概览 | 管理员 |
| F-ADM-02 | 用户管理 | 用户列表、新增、编辑、删除 | 管理员 |
| F-ADM-03 | 品种管理 | 宠物品种的增删改查 | 管理员 |
| F-ADM-04 | 宠物管理 | 宠物信息的发布、编辑、删除（含多图上传） | 管理员 |
| F-ADM-05 | 申请审核 | 审核领养申请：通过/拒绝（附审核意见） | 管理员 |
| F-ADM-06 | AI 审核辅助 | 查看 AI 对领养申请的辅助审核建议 | 管理员 |
| F-ADM-07 | 文件上传 | 上传宠物图片等文件 | 管理员 |

## 四、非功能需求

| 类别 | 要求 |
|------|------|
| **架构** | 前后端分离，前端 Vue3 SPA，后端 Spring Boot REST API |
| **认证** | JWT 无状态认证，Token 有效期合理设置 |
| **响应式** | 前端适配桌面端和移动端基础浏览 |
| **性能** | 首页加载 < 3 秒，API 响应 < 1 秒 |
| **安全** | 密码 BCrypt 加密，JWT 签名验证，权限中间件 |
| **兼容性** | 支持 Chrome、Edge、Firefox 最新版本 |
| **可用性** | 前端 UI 直观友好，错误提示明确 |

---

## 五、技术约束

1. **后端**：基于现有 Spring Boot 项目改造，保持原有数据库结构和业务逻辑
2. **前端**：Vue 3 + Vite + Element Plus，全新构建
3. **数据库**：MySQL 8.0，保持现有 schema 不变
4. **API 规范**：RESTful 风格，统一 JSON 响应格式
5. **部署**：开发阶段前后端独立运行（端口 8080 和 5173）

## 六、数据字典（摘要）

### 用户表 (user)
| 字段 | 类型 | 说明 |
|------|------|------|
| id | BIGINT | 主键 |
