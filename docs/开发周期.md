## 五、风险与应对

| 风险 | 概率 | 应对策略 |
|------|------|----------|
| JWT 与 Spring Security 集成复杂 | 中 | 参考官方文档，必要时简化过滤器链 |
| Element Plus 组件 API 学习成本 | 低 | 边查文档边开发，复用常见模式 |
| AI 接口不稳定影响匹配功能 | 低 | AI 匹配失败时降级为本地规则匹配 |
| 图片上传前后端格式不一致 | 中 | 统一使用 FormData + multipart/form-data |
| 前后端联调数据格式差异 | 中 | 阶段七预留充足调试时间（4h） |

## 六、开发规范

1. **Git 提交**：每完成一个阶段进行一次 commit，commit message 格式：`feat(scope): description`
2. **代码风格**：前端 ESLint + Prettier，后端遵循 Java 标准命名规范
3. **API 命名**：RESTful 风格，资源名用复数，操作通过 HTTP 方法表达
4. **组件命名**：Vue 文件使用 PascalCase，组件名与文件名一致
5. **错误处理**：所有 API 调用必须有 try-catch 或 .catch 处理
